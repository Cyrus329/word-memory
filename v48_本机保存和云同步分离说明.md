# v48 本机保存和云同步分离说明

本版修复云同步失败时用户误以为“保存不了”的问题：

- “只保存本机”不再请求 Supabase，直接保存到浏览器本机。
- 新增“尝试云同步”按钮：需要云端时再手动尝试。
- Supabase schema cache / RPC 函数缺失 / upstream 超时等错误不会影响本机保存。
- 云端请求超时从 20 秒缩短到 8 秒，减少卡顿。
- 保留 v47 的上一个单词释义提示、v46 的撤回上一个、v45 的语音修复。

当前截图里的 `Could not query the database for the schema cache` 通常表示 Supabase 数据库函数没有建好或没有刷新缓存，不是单词数据坏了。
