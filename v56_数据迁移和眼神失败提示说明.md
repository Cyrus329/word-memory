# v56 数据迁移和眼神失败提示说明

## 这版解决什么

1. 顶部按钮改成明显的：
   - 导入旧数据
   - 导出备份

2. 眼神翻词失败时提示更准确：
   - 可能是浏览器没给摄像头权限
   - 可能是当前不是 https 网页
   - 可能是眼神识别库 CDN 加载失败
   - 不影响普通背词和保存

## 数据为什么不在新网址上

浏览器的本机数据按网址隔离保存。

D 盘本地文件、旧 GitHub Pages、新 GitHub Pages 是不同位置，数据不会自动互通。

## 正确迁移方法

1. 打开有旧数据的旧页面。
2. 点“导出备份”，下载 JSON 文件。
3. 打开新的 GitHub Pages 网页。
4. 点“导入旧数据”，选择刚才的 JSON 文件。
5. 选择替换或合并。

## 云同步

如果 Supabase 里 `select now();` 都运行失败，说明云端数据库不可用。网页包无法修复 Supabase 数据库本身，需要等待恢复或新建 Supabase 项目。
