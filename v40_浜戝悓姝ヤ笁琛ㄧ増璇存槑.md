# v40 云同步三表版

这版按三张表保存云端数据：

| 表 | 用途 |
| --- | --- |
| `word_cloud_decks` | 词库信息：公开编号、名称、密码、是否公开 |
| `word_cloud_items` | 单词正文：word、音标、词性、意思、分组等 |
| `word_cloud_progress` | 学习进度：熟悉度、下次复习、重点、完成状态、总学习时长 |

## 解决的问题

1. 不再把 1800+ 个单词和学习记录混在一个大 JSON 里保存。
2. 手机端点“保存并开启同步”时会分批上传，降低超时概率。
3. 后续背单词、记完、忘记、重点、学习时长变化时，只同步 `word_cloud_progress`，不会反复上传整份词库正文。
4. 修复旧版报错：`Could not find the function save_word_memory_cloud(...) in the schema cache`。SQL 末尾已加入 `notify pgrst, 'reload schema';`。

## 必须做的步骤

1. 打开 Supabase。
2. 进入 SQL Editor。
3. 复制 `supabase-setup.sql` 全部内容运行一次。
4. 等 30-60 秒。
5. 把本文件夹全部上传覆盖 GitHub 仓库根目录。
6. 手机访问网页，地址后面加 `?v=40`。
7. 打开“云同步”，点“保存并开启同步”。

第一次保存会上传词库正文和进度，稍慢。以后自动保存只同步学习进度和时长。
