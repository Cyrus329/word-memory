# v41 数量对应修正说明

本版修正 v40 中分组数量显示不对应的问题。

原因：旧程序按英文单词去重，截图新导入的词如果以前词库里已经出现过，会被合并到旧分组，导致页面显示：

- 四级 5 显示 40，而不是 50
- 蓝色森林 8 显示 46，而不是 50

本版处理：

- 对本次截图导入的【四级 5】、【蓝色森林 8】、【蓝色森林 9】按独立 ID 保留。
- 即使同一个英文以前出现过，只要来源不同，也保留在这次对应分组里。
- 分组数量应与导入报告一致：四级 5 = 50，蓝色森林 8 = 50，蓝色森林 9 = 163。

上传 GitHub 后，手机端如果还显示旧数量，刷新一次或清理小程序/浏览器缓存后再打开。
